# Text Classification Pipeline

Binary classifier that predicts whether a PubMed abstract is associated  
with a protein of interest, using three model variants:

| Model | Vectoriser | Notes |
|---|---|---|
| **LR + TF-IDF** | TF-IDF | Best overall; used for default prediction |
| **LR + Word2Vec** | Word2Vec mean-pooling | Captures semantic similarity |
| **NB + TF-IDF** | TF-IDF | Fast baseline |

## Quick start

### Step 1 – Preprocess

```bash
python src/preprocess.py --config config/config.yaml
```

- Reads batched negative files (`data/raw/negative_df*`) and positive file  
  (`data/raw/positive_df`).  
- Removes overlapping PMIDs, assigns labels (0 / 1).  
- Runs the full NLP pipeline (lowercase → HTML strip → punctuation removal →  
  stopword removal → POS-aware lemmatisation) on `Abstract`, `Title`, and `Function`.  
- Writes `data/processed/merged_processed.tsv`.

### Step 2 – Train

```bash
python src/train.py --config config/config.yaml
```

- Trains all three model variants.  
- Prints per-model `classification_report`, confusion matrix, and AUC.  
- Saves model artefacts to `output/models/`.  
- Saves combined ROC-curve plot to `output/plots/roc_curves.png`.  
- Saves AUC summary to `output/metrics_summary.tsv`.

### Step 3 – Predict on unlabelled data

```bash
# input already has 'clean_text_Abstract' column (pre-processed):
python src/predict.py --config config/config.yaml --input data/new_records.tsv

# input has raw 'Abstract' column (pipeline applied on the fly):
python src/predict.py --config config/config.yaml \
                      --input data/new_records.tsv \
                      --raw

# use a different model:
python src/predict.py --config config/config.yaml \
                      --input data/new_records.tsv \
                      --model lr_w2v    # lr_tfidf | lr_w2v | nb_tfidf
```

Output: `output/predictions/<basename>_predictions_<model>.tsv`  
with two new columns: `predicted_label` (0/1) and `predicted_prob` (float).

### Step 4 – Evaluate on labelled data

Use this when you have a held-out dataset with known labels and want a full
performance report.

```bash
# evaluate the default model (LR + TF-IDF):
python src/evaluate.py --config config/config.yaml --input data/test_labelled.tsv

# input has raw 'Abstract' column (pipeline applied on the fly):
python src/evaluate.py --config config/config.yaml \
                       --input data/test_labelled.tsv \
                       --raw

# evaluate a different model:
python src/evaluate.py --config config/config.yaml \
                       --input data/test_labelled.tsv \
                       --model lr_w2v    # lr_tfidf | lr_w2v | nb_tfidf
```

**Input requirements:** TSV with a `Label` column (0 = negative, 1 = positive)
and the text feature column (or raw `Abstract` with `--raw`).

**Output:**
- Full classification report, confusion matrix, AUC, sensitivity, and specificity printed to stdout.
- ROC curve plot → `output/plots/eval_roc_<model>_<timestamp>.png`
- Confusion matrix plot → `output/plots/eval_cm_<model>_<timestamp>.png`
- Per-sample predictions with a `correct` flag → `output/predictions/eval_<basename>_<model>_<timestamp>.tsv`
- Scalar metrics appended to `output/metrics_summary.tsv`

## Configuration

All paths and hyperparameters live in `config/config.yaml`.  
No hard-coded values exist in the source files.

Key settings:

```yaml
model:
  test_size: 0.2            # train/test split ratio
  logistic_regression:
    C: 10                   # regularisation strength
  word2vec:
    vector_size: 100        # embedding dimensionality
    window: 5               # context window
```
